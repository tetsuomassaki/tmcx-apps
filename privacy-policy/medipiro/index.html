MediPiro プライバシーポリシー

最終更新日: 2026年5月7日

本プライバシーポリシーは、MediPiro（以下「本アプリ」）に適用されます。本アプリは、医療現場・薬局・調剤現場での患者ラベル印刷を支援する iOS アプリで、QRコード読み取りまたは手入力した患者情報を、同一 Wi-Fi ネットワーク上の Brother ラベルプリンタへ印刷するために使用されます。

データの収集について

本アプリは、個人データを外部サーバーに一切送信しません。具体的には:

  • 外部サーバーへの送信なし — 患者情報・操作ログ・端末識別情報を外部サーバーへ一切送信しません。
  • アナリティクス・トラッキングなし — Firebase / Google Analytics 等のアナリティクス SDK やトラッキングツールは含まれていません。
  • 広告なし — 広告の表示や広告主へのデータ共有は行いません。
  • クラウドサービスなし — 開発者が運営するクラウドサービスへの接続は行いません。iCloud 同期も使用しません。
  • 第三者提供なし — いかなる第三者にもデータを提供しません。

端末に保存されるデータ

以下のデータは、お使いの iPhone 内にのみ保存されます:

  • 患者情報の入力履歴（氏名、ID、関連メモ等） — Core Data (SQLite) に保存
  • 印刷履歴（印刷日時、ラベル内容） — Core Data に保存
  • アプリ設定（プリンタ IP アドレス、対応機種・用紙の選択、生体認証の有効/無効等） — UserDefaults に保存

すべてのデータは iOS Data Protection により、端末ロック時は暗号化されます。

ローカルネットワーク通信について

本アプリは、患者ラベルを印刷するために同一 Wi-Fi ネットワーク上の Brother ラベルプリンタとのみ通信を行います。

  • 検出方式: Bonjour（mDNS）によるサービス検出（_pdl-datastream._tcp / _printer._tcp / _ipp._tcp）
  • 印刷送信: TCP 9100 ポート（HP JetDirect / Raw 9100 規格）でラベルデータをプリンタへ直接送信
  • 送信先: ユーザーが選択または指定した Brother プリンタの IP アドレスのみ
  • インターネット送信なし: 上記通信はすべて LAN 内で完結し、インターネット経由で外部サーバーへ送信されることは一切ありません。

ローカルネットワーク権限は、上記プリンタ通信のためにのみ使用します。

カメラ権限について

本アプリはカメラ権限を要求します。カメラは患者情報の QR コード読み取りのみに使用されます。撮影した画像はファイルとして保存されず、QR コードのデコード処理にのみ使用されます。

生体認証（Face ID / Touch ID）について

本アプリは、印刷履歴の閲覧および全消去操作を保護するため、生体認証を要求する場面があります。生体認証データ（顔・指紋）は iOS のセキュアエンクレーブ内でのみ処理され、本アプリが生体認証データそのものにアクセスすることはありません。アプリは認証の成否のみを受け取ります。

Brother Print SDK について

本アプリは、Brother Industries, Ltd. が提供する Brother Print SDK V4.13.0 を組み込んでいます。同 SDK は、ユーザーが選択した Brother プリンタへラベルデータを送信するために使用され、本 SDK 自体が外部サーバー（Brother のクラウドや解析サービス等）と通信することはありません（ローカルプリンタへの直接通信のみ）。

医療データに関する免責事項

本アプリは医療機器ではなく、診断・治療・処方等の医療行為を行うものではありません。本アプリはあくまで患者識別ラベルの印刷を支援するツールであり、本アプリに表示される情報の正確性・完全性について保証するものではありません。医療上の判断は必ず担当の医師または医療従事者にご相談ください。

患者情報の取り扱いに関する利用者への注意

本アプリは患者情報をラベルとして印刷する性質上、入力された患者氏名・ID 等は端末内に保存されます。利用にあたっては、以下の点にご留意ください:

  • 端末を共用する場合は、画面ロックおよび本アプリの生体認証ロックを必ず有効にしてください。
  • 印刷履歴に保存された情報は、定期的に「全消去」機能で削除することを推奨します。
  • 端末を破棄・譲渡する際は、必ず本アプリをアンインストールするか、端末を初期化してください。

お子様のプライバシー

本アプリは医療従事者向けの業務支援アプリであり、13 歳未満のお子様の利用は想定していません。本アプリは個人情報を外部に送信することはありません。

ポリシーの変更

本プライバシーポリシーは随時更新される場合があります。変更があった場合は、このページに更新日とともに掲載します。

Privacy Policy (English)

This privacy policy applies to MediPiro ("the App"), an iOS app that assists medical facilities, pharmacies, and dispensaries in printing patient identification labels. The App reads patient information from QR codes or manual input and prints it to a Brother label printer on the same Wi-Fi network.

Data Collection: The App does not transmit any personal data to external servers. Specifically:

  • No external server transmission — Patient information, operation logs, and device identifiers are never sent to any external server.
  • No analytics or tracking — No analytics SDKs (Firebase, Google Analytics, etc.) or tracking tools are included.
  • No advertising — No ads are shown and no data is shared with advertisers.
  • No cloud services — No connection to developer-operated cloud services. iCloud sync is not used.
  • No third-party data sharing — Data is never provided to any third party.

Data Stored on Your Device: Patient input history (name, ID, related notes), print history (timestamp, label content), and app settings (printer IP address, supported printer/paper selection, biometric on/off) are stored in Core Data (SQLite) and UserDefaults. All data is encrypted via iOS Data Protection when the device is locked.

Local Network Communication: The App communicates only with a Brother label printer on the same Wi-Fi network in order to print patient labels. Discovery uses Bonjour (mDNS) services (_pdl-datastream._tcp, _printer._tcp, _ipp._tcp); printing sends label data directly to the printer's TCP port 9100 (HP JetDirect / Raw 9100). All communication stays within the local network and is never relayed to an external server.

Camera Permission: The App requires camera access solely for scanning QR codes that contain patient information. No images are stored as files; the camera feed is used only for QR code decoding.

Biometric Authentication: The App requests biometric authentication to gate access to print history and the bulk-delete operation. Biometric data (face / fingerprint) is processed exclusively within iOS's Secure Enclave; the App never has access to the raw biometric data and only receives the authentication result.

Brother Print SDK: The App embeds the Brother Print SDK V4.13.0 provided by Brother Industries, Ltd. The SDK is used solely to send label data to a Brother printer selected by the user; the SDK itself does not communicate with any external server (such as Brother's cloud or analytics services).

Medical Disclaimer: The App is not a medical device and does not provide any medical service (diagnosis, treatment, or prescription). The App is a label printing utility, and the accuracy or completeness of displayed information is not guaranteed. Always consult your physician or healthcare professional for medical decisions.

User Responsibilities Regarding Patient Information: Always enable device passcode and the App's biometric lock when the device is shared. Periodically delete print history using the bulk-delete feature. When disposing of or transferring the device, uninstall the App or factory-reset the device.

Children's Privacy: The App is intended for medical professionals and is not directed at children under 13. The App does not collect any personal information for external transmission.

Policy Changes: This privacy policy may be updated. Any changes will be posted on this page with a revised date.
